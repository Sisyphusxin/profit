## 1. 一句话定位

访客离开时应带走的主印象：**「这个人有意思；以后做文化、艺术、创新类项目或事情时，可以找她。」**

站点气质：**策展感**；整体 **简洁、宁静**；避免用时间线制造「必须常更新」的压力。

---

## 2. 信息架构（IA）

### 2.1 页面与动线

| 页面 | 职责 |
|------|------|
| **首页** | 首屏：简介 + **联系方式固定于此**；下列 **精选·发现（3）**、**精选·创造（3）**；各块 **「查看更多」** → 对应**独立列表页**。以**代表作**与**我推荐的**为亮点，不要求「站内动态条」。 |
| **发现**（独立页） | 「希望更多人知道」的全量归档（书影音、展览、文章、链接等）。 |
| **创造**（独立页） | 「自我表达」的全量归档（视频、文字、绘画、App…）；**媒介用标签**，非一级导航分裂。 |
| **关于** | **暂缓**（DEC-006）；叙事与版式待定后再补。 |

### 2.2 导航文案

- 栏目：**发现**、**创造**；英文 tag 规范见 DEC-007。  
- **关于**：当前不在主导航出现。

## 3. 内容模型（四字段 · DEC-005）

**发现 / 创造共用同一套条目结构**。

| 字段 | 说明 |
|------|------|
| **标题** | 条目名称。 |
| **一句话** | **发现**：为何推荐；**创造**：是什么、为何值得看。 |
| **链接** | 出口 URL。 |
| **一张图** | 视觉锚点；建议后续统一占位比例以利于排版（实现阶段定）。 |

实现层面可抽象为同一「卡片数据结构」，用栏目或标签区分「发现 / 创造」；首页条目可再打「精选」标记以便轮换展示。

---

## 4. 发现 vs 创造（意图）

- **发现**：你看到**好的**，希望**更多人知道**（策展）。  
- **创造**：你的**自我表达**；媒介只是表达方式。  

边界含糊时：问「主要是在帮好东西被看见，还是在说自己」——比在分类上僵持更快。

---

## 5. 英文 Tag 视觉（DEC-007）

- 文本：**`DISCOVER`**、**`CREATE`**（全大写）。  
- 相对中文标题：**明显更小一档**；颜色可再弱一档。  
- **字距偏宽**。  
- 全站同一种处理，不与「首字母大写」混用。

## 6. 仓库实现快照（对齐 DEC）

- **静态多页**：`index.html`（橱窗首页）、`discover.html`、`create.html`。  
- **样式**：共用 `css/styles.css`；占位图 `images/card-placeholder.svg`；头像 `images/avatar.jpg`（可选）。  
- **卡片**：整块 `<a>` 出站；字段见 §3（当前为手写 HTML，可日后改为 JSON/模板）。  
- **站内主导导航**：首页 · 发现 · 创造（无「关于」）。

### 备忘：与历史单页稿的关系

早期曾为 **单文件 `index.html`** 实验。**当前已实现多页 + 橱窗首页 + 列表页 + 站内导航**；若以子路径部署需注意相对路径一致性。
